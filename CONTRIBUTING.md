# 项目规范说明（ai-tanchishe）

## 项目结构
- snake.py 游戏主程序
- test_snake.py 单元测试及自动化测试
- README.md 使用说明与安装指引
- CONTRIBUTING.md 项目开发规范
- .venv/ 虚拟环境

## 代码规范
- 全部采用 PEP8 风格（自动工具格式化）
- 注释应中文或英文，简明准确
- 变量、函数命名清晰见义知意
- 测试必须覆盖全部关键逻辑

## 分支规范
- 所有功能开发需通过 feature/xxx 分支，禁止直接修改主分支
- 完成后统一通过 PR 合并至主分支

## commit 规范
- 使用清晰动词（feat/fix/docs/test/chore/refactor），说明变更内容
- 推荐英文，核心需求可中英结合

## 测试集要求
- 单元测试需覆盖90%以上业务逻辑
- 测试文件 test_*.py 命名

## 文档要求
- 所有新增功能/变更需及时补充/更新文档

## 贡献流程
1. fork/clone 仓库
2. 拉取 feature 分支开发
3. 提交 PR，附上需求说明/issue链接
4. 确认通过后合并

有任何疑问请通过 Issue 沟通
